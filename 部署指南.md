# 商品销售系统 - 部署指南

## 一、环境准备

### 1.1 软件环境要求
- **JDK**: 17 或更高版本
- **MySQL**: 8.0 或更高版本
- **Maven**: 3.6+ (项目已包含 Maven Wrapper，可无需安装)
- **浏览器**: Chrome、Firefox、Edge 等现代浏览器

### 1.2 检查环境

```bash
# 检查 Java 版本
java -version

# 检查 Maven 版本（可选）
mvn -version
```

## 二、数据库配置

### 2.1 创建数据库

方式一：使用命令行
```bash
mysql -u root -p
```

在 MySQL 命令行中执行：
```sql
CREATE DATABASE IF NOT EXISTS product_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

方式二：使用数据库管理工具
- Navicat
- MySQL Workbench
- DBeaver

创建名为 `product_db` 的数据库，字符集设置为 `utf8mb4`。

### 2.2 初始化数据表

执行 `src/main/resources/schema.sql` 文件：

方式一：命令行
```bash
mysql -u root -p product_db < src/main/resources/schema.sql
```

方式二：在 MySQL 客户端中直接运行 SQL 文件内容

### 2.3 验证数据

```sql
USE product_db;

-- 查看表
SHOW TABLES;

-- 应该看到 4 张表：category, product, orders, order_item

-- 查看分类数据
SELECT * FROM category;

-- 查看产品数据
SELECT * FROM product LIMIT 5;
```

## 三、应用配置

### 3.1 修改数据库连接

编辑 `src/main/resources/application.yml` 文件：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/product_db?useSSL=false&serverTimezone=UTC&characterEncoding=utf8&allowPublicKeyRetrieval=true
    username: root          # 修改为你的数据库用户名
    password: 123456        # 修改为你的数据库密码
```

### 3.2 可选配置

#### Redis 配置（可选）
如果需要使用 Redis 缓存：

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    database: 0
    timeout: 5000ms
```

#### Elasticsearch 配置（可选）
如果需要高级搜索功能：

```yaml
spring:
  elasticsearch:
    uris: http://localhost:9200
    connection-timeout: 5s
    socket-timeout: 10s
```

## 四、启动应用

### 方式一：使用启动脚本（推荐）

Windows 系统双击运行：
```
start.bat
```

该脚本会自动：
1. 检查 Java 环境
2. 下载 Maven 依赖
3. 启动应用

### 方式二：使用 Maven 命令

```bash
# Windows
mvnw.cmd spring-boot:run

# Linux/Mac
./mvnw spring-boot:run
```

### 方式三：打包后运行

```bash
# 编译打包
mvn clean package -DskipTests

# 运行 jar 包
java -jar target/product-sales-system-0.0.1-SNAPSHOT.jar
```

### 方式四：使用 IDE

在 IntelliJ IDEA 或 Eclipse 中：
1. 找到 `ProductSalesSystemApplication.java`
2. 右键 -> Run 或 Debug

## 七、常见问题排查

### 7.1 启动失败

**问题 1**: 数据库连接失败
```
错误信息：Communications link failure
解决方案:
1. 检查 MySQL 服务是否启动
2. 检查数据库连接配置是否正确
3. 检查防火墙设置
```

**问题 2**: 端口被占用
```
错误信息：Port 8080 was already in use
解决方案:
1. 修改 application.yml 中的端口号
   server:
     port: 8081  # 改为其他端口
2. 或者关闭占用 8080 端口的程序
```

**问题 3**: 找不到主类
```
错误信息：Could not find or load main class
解决方案:
1. 重新编译项目：mvn clean compile
2. 检查 JDK 版本是否为 17+
```

### 7.2 页面访问失败

**问题 1**: 404 错误
```
原因：静态资源路径配置错误
解决方案:
1. 确认 index.html 在 src/main/resources/static/目录下
2. 检查应用是否正常启动
3. 查看控制台日志是否有错误
```

**问题 2**: 跨域错误
```
原因：浏览器跨域限制
解决方案:
1. 项目已配置全局跨域，检查 CorsConfig.java
2. 确保前后端端口一致
```

### 7.3 功能异常

**问题 1**: 图片无法上传
```
原因：未配置文件上传
解决方案:
当前版本支持 Excel 导入，图片上传功能暂未实现
```

**问题 2**: 导入失败
```
原因：Excel 格式不正确
解决方案:
1. 使用提供的导入模板.csv 文件
2. 确保第一行为表头
3. 商品编号不能重复
```

**问题 3**: 图表不显示
```
原因：ECharts 库加载失败
解决方案:
1. 检查网络连接（使用 CDN 加载）
2. 可以下载 ECharts 到本地
3. 查看浏览器控制台错误信息
```

## 八、性能优化建议

### 8.1 数据库优化

```sql
-- 添加索引（已自动创建）
CREATE INDEX idx_product_code ON product(product_code);
CREATE INDEX idx_category_id ON product(category_id);
CREATE INDEX idx_order_date ON orders(order_date);
```

### 8.2 应用优化

1. **启用缓存**: 配置 Redis 缓存热点数据
2. **连接池优化**: 调整 HikariCP 参数
3. **分页大小**: 根据实际需求调整默认分页大小

### 8.3 前端优化

1. **CDN 加速**: 使用国内 CDN 镜像
2. **资源压缩**: 压缩 CSS 和 JS 文件
3. **懒加载**: 按需加载组件

## 九、生产环境部署

### 9.1 打包部署

```bash
# 生产环境打包
mvn clean package -DskipTests -Pprod

# 后台运行
nohup java -jar target/product-sales-system-0.0.1-SNAPSHOT.jar > app.log 2>&1 &
```

### 9.2 Docker 部署（可选）

创建 Dockerfile:
```dockerfile
FROM openjdk:17-jdk-slim
WORKDIR /app
COPY target/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

构建和运行:
```bash
docker build -t product-sales-system .
docker run -d -p 8080:8080 product-sales-system
```
